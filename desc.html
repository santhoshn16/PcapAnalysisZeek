Terminology
Name Description
TPC(Total Packet Count) Total number of packets that were seen in a single connection.
SPC(Small Packet Count) Number of packets that were seen with payload less than 20 bytes.
GPC(Gap Packet Count) Number of packets that were seen in between consecutive small packets. More gap packets means less continuous small packets.
DATAEXCHANGED(B) Sum of data (in bytes) that has been seen in a single connection.
RATIO(ser/cli) This is calculated based on how much data has been sent from the server divided by the amount of data sent by the client. This helps in knowing if any data exfiltration has happened or not.
METRIC T() Continuous small packets show that traffic is interactive in many services like ftp, telnet, http, ssh. T is calculated based on spc-gpc-1/tpc
METRIC ALPHA() Machine driven connection has a tendency of sending packets continuously with short intervals(here considered 2 sec) between them. So now we can use that to remove those types of connections from the observable list. Ratio of the above statement is calculated and a threshold is set after many considerations. (More no of consecutive sp fall in range of 2 sec, closer will be the ratio to 1)
